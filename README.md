## Reflection

### Briefly summarize the requirements and goals of the application you developed. What user needs was this application designed to address?

The goal of this project was to develop both a Console application and a WPF application that met the software requirements while demonstrating my ability to modify existing code and create new functionality. The application was designed to provide users with an easy way to interact with the program through clear prompts and a graphical interface. It focused on making information easy to access while ensuring the application was reliable, organized, and user-friendly.

### What did you do particularly well in developing this application?

One of my strengths during this project was carefully following the software requirements and ensuring the application functioned correctly after each change. I also focused on keeping my code organized and readable, making it easier to identify problems and implement additional features. Testing my work throughout development helped me catch issues before moving on to the next step.

### Compare and contrast the Console and WPF application designs. What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful?

The Console application relied on text-based prompts and user input, making it simple but requiring users to remember available options. The WPF application provided a graphical interface with buttons, labels, and other visual elements that made navigation more intuitive. The WPF version reduced the chance of user input errors because users could interact with clearly labeled controls instead of typing commands. Both versions met the same functional requirements, but the WPF application created a more user-centered experience by making the interface easier to understand and navigate.

### How did you approach the process of debugging and coding your application? What techniques or strategies did you use? How could you use those techniques or strategies in the future?

I approached debugging by testing the application after making small changes instead of waiting until the end of development. When errors occurred, I carefully read the error messages, checked the affected code, and verified that the program produced the expected output. Breaking problems into smaller pieces made them easier to solve. These strategies will continue to be useful in future software development projects because they help identify issues early and reduce the time needed to fix bugs.

### Where did you have to be innovative to overcome a challenge in the full application development process?

One challenge was adapting existing code to meet the project requirements while maintaining functionality. I had to think critically about how different sections of the application interacted and make changes without introducing new errors. By testing frequently and making incremental improvements, I was able to successfully complete the application while meeting the project requirements.
